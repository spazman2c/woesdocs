# Command Reference

IQ Connect commands use a consistent shape:

```text
<direction> <operation> <tenant-or-customer> <environment> [mode]
```

## Direction Conventions

| Prefix | Meaning |
| --- | --- |
| `from-*` | Pulls partner data into IQ Reseller, such as orders, returns, cancellations, or local replay files. |
| `to-*` | Sends IQ Reseller data outward, such as catalog, inventory, ASN, invoices, CRM data, or order updates. |
| `bestbuy service` | Service-style Best Buy/Symphony workflow command. |

## Best Buy Symphony

| Purpose | Command |
| --- | --- |
| Import Best Buy orders | `from-bestbuy so ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production` |
| Import Best Buy orders, delta mode | `from-bestbuy so ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta` |
| Send SO updates / ETA updates | `to-bestbuy soupdate ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta` |
| Manually resend SO updates / ETA updates | `to-bestbuy resendsoupdate ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta` |
| Resend ASN | `to-bestbuy resendasn ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta` |
| Send availability | `to-bestbuy avl ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta` |

For the Kudu command used to force missing PARTpoint promise date updates, see [Best Buy Promise Date Resend](/operations/best-buy-promise-date-resend).

## Execution Flow

Most commands follow this runtime sequence:

1. Resolve the active integration section from the command arguments.
2. Populate `CheatSheet` with integration, Azure, IQ API, and tenant settings.
3. Validate the tenant context.
4. Read source records through `IqReader` or a CRM-specific reader.
5. Format the partner-specific payload.
6. Send or stage the payload.
7. Update sync state and feed history.
