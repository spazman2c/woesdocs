# Support Runbook

Use this page when a customer reports a missing order, missing shipment, missing invoice, promise-date issue, or line-item mismatch.

## Common Cases

| Case | First Checks | Evidence To Collect |
| --- | --- | --- |
| Order not posted | Confirm inbound command, tenant context, mirror payload, and IQ API response. | Partner PO, mirror file, WebJob log, IQ sales order lookup. |
| Shipment not sent | Confirm unshipped SO query results, delta flags, ASN formatting, and SFTP delivery. | SO ID, shipment command, generated ASN, SFTP output folder. |
| Invoice not sent | Confirm uninvoiced SO query results, invoice command, and sync flags. | Invoice command, generated invoice payload, sync flag rows. |
| Best Buy promise date not sent | Compare `st.deliverydate` to `iqconnect_so.eta`, queue SOIDs in `iqconnect_batch_soupdate_resend`, and run the Kudu resend command. | Best Buy PO, IQ SOID, promise date, `voupdates` file, Kudu/WebJob output. |
| Incorrect line item number | Compare IQ line item comments against expected partner reference tokens. | SO line comments, partner PO line number, generated outbound payload. |

## Standard Execution Flow

1. Set the active section from command arguments.
2. Populate `CheatSheet` from app config and database records.
3. Load Azure, IQ API, and tenant settings.
4. Validate tenant context before touching partner data.
5. Run the inbound or outbound integration wrapper.
6. Preserve mirror copies and review sync flags after the run.

## Best Buy Promise Date Resend

Use [Best Buy Promise Date Resend](/operations/best-buy-promise-date-resend) when PARTpoint reports that SO promise date updates were not received by Best Buy.

The manual recovery flow is:

1. Map Best Buy PO numbers to IQ SOIDs.
2. Confirm `st.deliverydate` has the correct promise date.
3. Compare it to `iqconnect_so.eta`.
4. Insert affected SOIDs into `iqconnect_batch_soupdate_resend`.
5. Run the `resendsoupdate` command from Kudu CMD.
6. Confirm the resend queue clears and a new `Ppnt_*_voupdates.xml` file is produced.

## SPS Line-Item Comment Format

SPS stores partner line information in SO detail comments:

```text
[SPSLN]1[ENDLN][SPSQTY]1[ENDQTY][SPSPO]3888470OD00914[ENDPO][SPSDATE]2026-05-07[ENDDATE][SPSS]DS~88[ENDS][SPSITEM]2800070000SI[ENDITEM]
```

Use this when debugging SPS line-number or quantity mismatches.

## Evidence Checklist

| Evidence | Why It Matters |
| --- | --- |
| Original partner file | Confirms what the partner actually sent. |
| Mirror copy | Confirms what IQ Connect received or generated. |
| WebJob/Kudu log | Shows command arguments, record counts, and runtime errors. |
| Generated XML/CSV | Confirms what would have been sent to the partner. |
| `iqconnect_*` rows | Shows sync status, mappings, and resend queue state. |
| Source IQ rows | Confirms the current business data in IQ Reseller. |
