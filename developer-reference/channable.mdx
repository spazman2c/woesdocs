# Channable Developer Notes

For the operational workflow, field mapping, command, and troubleshooting checklist, start with [Channable](/integrations/channable).

## Main Components

| Component | Purpose |
| --- | --- |
| `GetDataFromVendor_Channable()` | Entry point used by the catalog command. |
| `PopulateCheetSheetFromDb()` | Loads Channable settings from database configuration. |
| `Uri_FullCatalog` | SQL/API query that builds the flat catalog feed. |
| `maxlist_items` | Marketplace listing table joined into the catalog query. |

## Developer Checks

| Area | What To Check |
| --- | --- |
| Missing item | Confirm the item exists in both the item master data and the MaxList records used by the catalog query. |
| Pricing | Verify the item pricing tier fields used by `sale_price1` through `sale_price8`. |
| Images | Confirm `imageurl` is populated for the item. |
