# Dashboards

## Monitoring Dashboards 

Three web application projects provide monitoring and operational visibility into IQConnect's sync operations. They share similar UI structures but target different frameworks and authentication models. 

### IqConnectCore (Primary Dashboard) 

Framework: ASP.NET Core with Razor Pages 
Authentication: Azure AD B2C (tenant: poigai.b2clogin.com ) 
Features: Sync status, WebJob management, feed log viewer, settings display 

 | View | Purpose 
| --- | --- |
 | Index | Dashboard landing — initializes CheatSheet from session 
 | Settings | Displays resolved CheatSheet configuration 
 | Sync | Sync page with feed status overview 
 | WebJobs | Lists Azure WebJob instances and their states 
 | Log | Feed history log from iqconnect_lastsync table 
Models: OpenOrdersModel, FulfilledOrdersModel, CancelledOrdersModel, ReturnedOrdersModel, FailedOrdersModel, LastSyncViewModel, WebJobListModel 

### IQConnectWeb / IQConnectWeb3 (Legacy) 

Framework: ASP.NET WebForms 
Purpose: Earlier versions of the monitoring dashboard. IQConnectWeb3 includes its own .sln file and appears to be a standalone variant. Both share the same page structure (Pages/, CommonClasses/) with the same Bootstrap/jQuery UI. 

### IqFeedApi 

Framework: ASP.NET Web API 2 
Purpose: Exposes feed data via REST API with auto-generated help documentation (HelpPage area). Contains HomeController and ValuesController. Used for programmatic access to feed data from external systems.
