# Integrations

## Integration Wrappers 

Each external system integration is encapsulated in a dedicated wrapper class within IqSync/ApiWrapper/ . These classes handle protocol translation, feed formatting, SFTP file management, and post-operation state tracking. 

### Active Integrations 

#### Best Buy Symphony Active 

Customer: PARTpoint 
File: Symphony.cs (6,305 lines) 
Protocol: SFTP (XML files) 
Feeds: Full bidirectional workflows — catalog, orders, ASN, invoices, returns, cancellations 

#### SPS Commerce Active 

Customer: OSI Global 
File: SpsCommerce.cs (3,861 lines) + SpsCommerceDS.cs (1,171 lines) + SpsFormatConverter.cs (362 lines) 
Protocol: SFTP (XML/JSON) 
Feeds: Orders, ASN, invoices, acknowledgements 

#### HubSpot Active 

Customers: DHD, OSI Global, + others 
File: HubSpot.cs (2,731 lines)

#### Channable Active

### Common Integration Pattern 

All active wrappers follow the same structural pattern: 
1. Constructor receives IqReader (pre-fetched IQ data) + feedType + CheatSheet (config)
2. Constructor calls PrepareFeed_[FeedType]() to transform IQ XML into target format
3. WriteFeed() / WriteSymphonyFeedToFile() writes to SFTP or posts to API
4. PostOpUpdateQuery is set with SQL to mark records as processed
5. Caller (IqSync Program.cs) executes PostOpUpdateQuery via VssLastMile
