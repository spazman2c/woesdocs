# Updates Overview

Updates are customer-facing product notes shown in the Woes support widget.
They are useful when a product change, docs update, incident resolution, or
feature release should be visible where customers already ask for help.

Updates are managed from the **Updates** workspace surface. They are not bulk
email, Discord announcements, or generic marketing broadcasts.

## What Updates Support

| Capability | Details |
| --- | --- |
| Widget delivery | Published updates appear in the widget Updates tab when the tab is enabled. |
| Targets | Updates can target the workspace widget surface or a specific agent/widget surface where configured. |
| Audience | Choose broad visibility or verified-customer visibility. |
| Content | Title, body, optional cover image, CTA label, and CTA URL. |
| Lifecycle | Draft, scheduled, published, archived, and expiry behavior. |
| Metrics | Per-update impressions and CTA clicks from widget events. |

## Good Update Use Cases

- A feature request has shipped.
- A known API issue has been resolved.
- Documentation or SDK examples were updated.
- Customers need a migration reminder.
- A workaround changed or is no longer required.

## Before Publishing

- Confirm the linked docs or release note is current.
- Remove private customer data and internal notes.
- Decide whether anonymous visitors or only verified visitors should see it.
- Confirm the widget Updates tab is enabled for the intended surface.
- Preview the update in the widget context where customers will see it.
