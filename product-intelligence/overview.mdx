# Product Intelligence

Product intelligence is the process of turning support activity into product
signals: recurring bugs, confusing API areas, feature requests, integration
blockers, and documentation gaps.

## Where Product Signals Come From

| Signal | Source |
| --- | --- |
| Repeated customer questions | Inbox conversations and AI handoffs. |
| Bugs and blockers | Issues created from conversations. |
| Feature requests | Tagged issues and customer feedback. |
| Docs gaps | Low-confidence answers and repeated clarifying questions. |
| Account impact | Client profiles and account-level issue history. |

## Recommended Loop

<Steps>
  <Step title="Capture the signal">
    Tag the conversation or create an issue when the work should be tracked.
  </Step>
  <Step title="Attach context">
    Include customer, account, source conversation, product area, and relevant links.
  </Step>
  <Step title="Group related work">
    Use tags, custom fields, and issue status categories to keep themes visible.
  </Step>
  <Step title="Review analytics">
    Look for trends by channel, source, survey result, and customer segment.
  </Step>
  <Step title="Close the loop">
    Update customers, improve docs, or rescan context when the product changes.
  </Step>
</Steps>
