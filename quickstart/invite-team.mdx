# Invite Your Team

Workspace admins can invite operators and configure what each member can read or change.

## Add Users

<Steps>
  <Step title="Open Settings → Users">
    Review current members, roles, and activity.
  </Step>
  <Step title="Invite a teammate">
    Add the teammate's email address and choose an initial role.
  </Step>
  <Step title="Assign permissions">
    Use the role defaults or customize permissions in **Settings → Users → Access**.
  </Step>
  <Step title="Review viewing state">
    The Users table can show what an operator is currently viewing and lets admins jump directly to that item when available.
  </Step>
</Steps>

## Default Role Guidance

| Role | Typical use |
| --- | --- |
| Owner | Full workspace setup, billing-adjacent operations, permissions, keys, and security settings. |
| Admin | Day-to-day configuration, channels, agents, automations, and team management. |
| Member | Inbox, issues, replies, notes, macros, and customer support work. |
| Viewer | Read-mostly or audit access. By default, Viewers can still send Team Chat messages. |

## Invite Acceptance

When an invitee opens the invitation link, Woes verifies the invitation and asks
for a short email code before account setup continues. The invitee then sets a
password of at least eight characters and joins the workspace with the role or
direct access assigned by the admin.

## Operator Onboarding

1. Read [Inbox](/platform/inbox) and [Issues](/platform/issues).
2. Learn how [Operator Takeover](/platform/inbox#operator-takeover) works.
3. Review [Agents](/agents/overview) so operators understand what the AI can and cannot answer.
4. Review [Data Privacy](/security/data-privacy) before using customer screenshots, logs, or credentials.

<Tip>
  Start new operators as Members unless they need to change channels, keys, automations, or workspace-wide settings.
</Tip>
