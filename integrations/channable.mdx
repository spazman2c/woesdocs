# Channable

Channable is the Renewtech catalog integration. IQ Connect builds a flat product feed from IQ Reseller item, inventory, pricing, category, and MaxList data, then publishes it for marketplace distribution through Channable.

## Feed Overview

| Feed | Direction | Customer | Purpose |
| --- | --- | --- | --- |
| `catalog` | IQ to Channable | Renewtech | Sends product catalog, pricing, stock, category, condition, and image data. |

The feed is generated from the `Uri_FullCatalog` query. That query joins `maxlist_items` with `item` so the outgoing catalog includes both marketplace-listing state and item master data.

## Catalog Fields

| Field | Source |
| --- | --- |
| `id` | Composite key: `UPPER(CONCAT(mfgr, '~~', itemnumber, '~~', conditioncode))`. |
| `brand` | Manufacturer (`mfgr`). |
| `title` / `item_group_id` | IQ item number. |
| `condition` | IQ condition code. |
| `description` | Item description. |
| `price` | Item price. |
| `sale_price1` through `sale_price8` | Tiered pricing from the item table. |
| `stock` | Available quantity. |
| `product_type` | Concatenated category hierarchy, up to four category levels. |
| `image_link` | Item image URL. |

## Invocation

```bash
to-channable catalog renewtech production reset
```

The command runs through `GetDataFromVendor_Channable()` and uses `PopulateCheetSheetFromDb()`, so settings are loaded from the database instead of only from `App.config`.

## Troubleshooting

| Symptom | First Checks |
| --- | --- |
| Item missing from feed | Confirm the item exists in the catalog query and has the required MaxList/item records. |
| Price looks wrong | Check the item pricing tiers used by `sale_price1` through `sale_price8`. |
| Stock looks wrong | Check available quantity and MaxList listing state. |
| Category missing | Check category assignments on the item record. |
| Image missing | Check `imageurl` on the item record. |
