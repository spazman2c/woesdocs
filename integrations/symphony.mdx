# Best Buy Symphony

Best Buy Symphony is the largest IQ Connect fulfillment integration. It connects PARTpoint's IQ Reseller tenant to Best Buy over XML files and SFTP, covering the full order lifecycle from catalog publication through PO import, acknowledgements, ETA updates, ASN, invoices, returns, and cancellations.

## Supported Feeds

| Feed | Direction | Description |
| --- | --- | --- |
| `itemsku` | IQ to Best Buy | Master item catalog: SKU, manufacturer, description, UPC, and related item data. |
| `bom` | IQ to Best Buy | Part associations / bill of materials. |
| `subs` | IQ to Best Buy | Item substitutions. |
| `avl` | IQ to Best Buy | Inventory availability by quantity and condition. |
| `price` | IQ to Best Buy | Pricing updates. |
| `so` | Best Buy to IQ | Vendor orders / purchase orders from Best Buy. |
| `soupdate` | IQ to Best Buy | Order acknowledgements, ETA updates, and vendor order updates. |
| `shipment` | IQ to Best Buy | ASN shipment notices with tracking. |
| `invoice` | IQ to Best Buy | Vendor invoices. |
| `returns` | Best Buy to IQ | Return requests. |
| `cancel` | Best Buy to IQ | Cancellation requests. |
| `returnack` | IQ to Best Buy | Return and cancellation acknowledgements. |

## Key Components

| Component | Purpose |
| --- | --- |
| `IqSync/ApiWrapper/Symphony.cs` | Main Best Buy wrapper. Builds XML, manages feed-specific logic, and writes outbound files. |
| `SymphonyReturnCancel.cs` | Return/cancel acknowledgement data structures. |
| `Uhaul` | Handles file staging and transfer between Azure and SFTP paths. |
| `iqconnect_so` | Tracks Best Buy PO to IQ SO mapping, workflow status, feed GUID, and last ETA sent. |
| `iqconnect_posted_voupdates` | Tracks posted vendor order update statuses. |
| `iqconnect_batch_soupdate_resend` | Manual resend list for SO update / ETA recovery. |

## SFTP Configuration

| Setting | Value |
| --- | --- |
| Input folder | `/from_bby` for inbound POs, returns, and cancels. |
| Output folder | `/to_bby` for outbound catalog, ASN, invoices, and updates. |
| File prefix | `Ppnt` for PARTpoint files. |
| Transport | SFTP via Best Buy managed file transfer. |

## Promise Date / ETA Updates

PARTpoint updates the SO promise date in IQ Reseller. IQ Connect reads that date from `st.deliverydate` and sends it to Best Buy in the `voupdates` file as `ExpectedActivityDate`.

For manual recovery when promise dates were entered but Best Buy did not receive the update, use [Best Buy Promise Date Resend](/operations/best-buy-promise-date-resend).

## Common Commands

```bash
from-bestbuy so ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production
from-bestbuy so ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta

to-bestbuy soupdate ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta
to-bestbuy resendsoupdate ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta
to-bestbuy resendasn ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta
to-bestbuy avl ec87ce03-5eaf-42ae-b9bf-9ad88e2a2cbf production delta
```

## Operational Notes

| Behavior | Detail |
| --- | --- |
| Partial voids | The `BBYVOID` marker is used when a line item is voided without cancelling the full SO. |
| Invalid line numbers | Line number `99999` is treated as malformed and filtered from ASN feeds. |
| Feed history cleanup | Catalog feed content is purged after roughly 3 days; order feed content after roughly 42 days. |
