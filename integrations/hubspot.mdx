# HubSpot

HubSpot is the CRM integration layer for IQ Connect. It pushes IQ Reseller companies, contacts, sales pipeline records, deal updates, line items, and gross margin data into HubSpot. It also receives accepted HubSpot quotes through webhooks and creates IQ sales orders through an Azure Durable Functions flow.

## What It Handles

| Direction | Feed | HubSpot Object | Purpose |
| --- | --- | --- | --- |
| IQ to HubSpot | `company` | Companies | Sends CV customer/vendor records that have not been synced yet. |
| IQ to HubSpot | `contact` | Contacts | Sends contacts with valid emails and tracks duplicate exclusions. |
| IQ to HubSpot | `sp` | Deals | Creates and updates sales pipeline deals, including line items, stages, reps, amounts, and dates. |
| IQ to HubSpot | `spupdate` | Deals | Detects IQ SP status changes and patches existing HubSpot deals. |
| IQ to HubSpot | `spupdateli` | Deal line items | Updates deal line item details. |
| IQ to HubSpot | `sogm` | Deals | Sends gross margin data for synced deals. |
| HubSpot to IQ | Quote webhook | Sales order | Creates IQ sales orders from accepted HubSpot quotes. |

## Quote-To-Order Flow

1. HubSpot quote is accepted.
2. HubSpot posts a webhook payload to `IqConnectWebhooksAsync`.
3. The Azure Durable Function reads the tenant from the `iq_tenant` value.
4. The function retrieves the full quote from the HubSpot API.
5. IQ Connect creates the sales order through the IQ API.
6. Sync tables are updated so later deal, line-item, and gross-margin jobs can continue from the same record.

## Tenant Configuration

HubSpot settings are tenant-specific and stored in `integration_settings`. Common settings include:

| Setting | Purpose |
| --- | --- |
| `dealstage_open` | HubSpot stage used for active/open deals. |
| `dealstage_inactive` | HubSpot stage used when an IQ opportunity becomes inactive. |
| `hubspotid_company` | IQ field that stores or maps the HubSpot company ID. |
| `sp_dealtype` | IQ field used to map deal type values. |
| `ordering_contact` | Contact mapping used when creating or updating deals. |

Deal stage metadata is also loaded from `hubspotdealstage`, including the HubSpot internal name and probability/percentage.

## Key Tables

| Table | Purpose |
| --- | --- |
| `iqconnect_crm_company` | IQ CV to HubSpot company mapping. |
| `iqconnect_crm_contact` | IQ contact to HubSpot contact mapping. |
| `iqconnect_crm_duplicate_contacts` | Contacts excluded from sync to avoid duplicates. |
| `iqconnect_crm_sp` | IQ SP to HubSpot deal mapping and status tracking. |
| `iqconnect_syncflags` | Used by some jobs to prevent repeat processing, such as gross margin sync. |
| `integration_settings` | Per-tenant HubSpot settings. |

## Troubleshooting

| Symptom | First Checks |
| --- | --- |
| Company or contact did not sync | Confirm the source record is eligible, has required fields, and is not already mapped in the CRM tracking table. |
| Deal did not update | Check `iqconnect_crm_sp`, deal stage mappings, and the `spupdate` job log. |
| Gross margin did not send | Check the gross margin query and whether `iqconnect_syncflags.grossmargin_done` already marks the SO as handled. |
| Quote did not create an SO | Check the webhook payload tenant, Azure Function run history, HubSpot API response, and IQ API response. |
