# Woes CLI

The Woes CLI is the first-party terminal app for working with your workspace
from a shell. It is useful for support operators who want a fast inbox view and
developers who prefer command-line access to conversations, issues, and client
records.

## Status

**Available now.** Copy the install command from **Workspace Settings ->
Integrations -> Woes CLI**, or run it directly:

```bash
curl -fsSL https://woes.dev/install.sh | sh
```

Then authorize the terminal:

```bash
woes login
```

## What It Does

| Capability | What you can do |
| --- | --- |
| Terminal inbox | Run `woes` to open a full-screen inbox for open and waiting conversations. |
| Conversation review | Open a selected conversation and read its transcript. |
| Issue review | List and inspect workspace issues from the terminal. |
| Client review | List and inspect client account aggregates. |
| Script output | Add `--json` for machine-readable output. |
| Local logout | Run `woes logout` to remove the local CLI credential. |

## Setup

<Steps>
  <Step title="Copy the install command">
    Open **Workspace Settings -> Integrations -> Woes CLI** and click
    **Copy install**.
  </Step>
  <Step title="Install">
    Paste the command into a terminal on a machine with Node.js 18 or newer.
  </Step>
  <Step title="Authorize">
    Run `woes login` and approve terminal access in the browser.
  </Step>
  <Step title="Open the inbox">
    Run `woes` to enter the full-screen terminal inbox.
  </Step>
</Steps>

## Useful Commands

```bash
woes
woes status
woes inbox list
woes conversations view <conversation-id>
woes issues list --priority high
woes clients list --search acme
woes logout
```

## Current Boundaries

- The CLI does not replace the browser operator inbox.
- The CLI does not currently send operator replies, run macros, assign
  conversations, or change AI takeover state.
- CLI access is scoped to the workspace you authorize.
- Treat the local machine as trusted. Log out when a device should no longer
  have terminal access.
