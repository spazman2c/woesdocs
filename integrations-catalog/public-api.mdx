# Public API

The Woes Public API is the server-side integration surface for conversations, issues, clients, agents, sources, definitions, usage, and analytics.

## Status

**Available now.** Public REST endpoints live under `/api/v1` and authenticate with workspace API keys that start with `woesk_`.

## What it does

The Public API lets backend integrations:

- List, create, and update support conversations.
- Add messages to existing conversations without invoking the widget runtime.
- Create and read issues.
- Seed or inspect client/account records.
- Read public agent metadata and source inventory.
- Read usage and sanitized analytics aggregates.

## Where it appears in Woes

- **Settings -\> Keys** creates and revokes workspace API keys.
- **Settings -\> Billing** tracks current-month public API calls by key prefix.
- **Settings -\> Integrations** shows whether API keys are ready.
- **API Reference** contains endpoint-by-endpoint examples and generated schemas.

## Key handling

Workspace API keys are private server-side secrets. The full key is shown once at creation and stored hashed after that. Do not put `woesk_` keys in widget snippets, browser JavaScript, mobile apps, public docs, screenshots, or customer-visible messages.
