# Woes MCP

Woes MCP lets AI tools and internal assistants inspect Woes support data through the remote Model Context Protocol endpoint.

## Status

**Available now.** The MCP endpoint is available at `https://woes.dev/api/mcp` and uses workspace API keys for authenticated tool calls.

## What it does

Woes MCP is useful when an AI assistant needs controlled access to workspace support data, such as:

- Listing recent conversations and issues.
- Creating issues from an internal escalation workflow.
- Reading clients, agents, sources, usage, and sanitized analytics.
- Building daily support briefs or agent rollout checks.

## Where it appears in Woes

- **Settings -\> Integrations** shows Woes MCP readiness based on active workspace API keys.
- **Settings -\> Keys** creates the API key used by the MCP client.
- **API Reference -\> MCP Server** documents tool names, scopes, and JSON-RPC examples.

## Security model

MCP tool calls use the same public API controls as REST integrations: workspace API key auth, key scopes where available, rate limits, usage tracking, tenant isolation, and redacted response shapes. The MCP server does not expose raw secrets, service-role data, operator-only traces, provider internals, or cross-workspace records.
