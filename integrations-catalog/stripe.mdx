# Stripe Billing

Stripe is the billing integration Woes uses for workspace subscriptions, plan changes, add-ons, and subscription management.

## Status

**Available now.** Workspaces can use Settings -\> Billing to manage plans and Stripe-backed subscription flows where Stripe is configured.

## What it does

Stripe billing helps admins:

- See the current Woes plan and subscription status.
- Start plan changes through Stripe Checkout.
- Open Stripe Billing Portal through **Manage subscription**.
- Add supported capacity such as operator seats or extra agent/widget capacity.
- Keep Woes plan limits in sync with Stripe webhooks.
- Track public API calls and live API checks against plan capacity.

## Where it appears in Woes

- **Settings -\> Billing** is the main billing surface.
- **Settings -\> Integrations** shows Stripe billing status as part of the workspace integration catalog.
- **Settings -\> Keys** creates API keys whose usage is counted in billing usage meters.

## Internal boundary

This page describes Woes billing through Stripe. It is different from a customer-owned Stripe data connector for importing subscription context into support conversations. Customer-owned billing-context integrations may be designed separately later and should not be assumed to be live from this page.
