# PagerDuty Candidate

PagerDuty is a future integration candidate for support escalation into incident and on-call workflows.

## Status

**Future candidate.** This is not currently a live Woes connector.

## What it could do

A PagerDuty integration could connect urgent support work to the right responder:

- Create or trigger PagerDuty incidents from Woes automations.
- Escalate SLA breaches, outage reports, or high-priority customer issues.
- Include customer impact, channel, priority, transcript summary, and source evidence.
- Write incident links back onto Woes issues for operator visibility.
- Help close the loop when the incident is acknowledged or resolved.

## Why it fits Woes

Developer support sometimes reveals production-impacting issues before monitoring does. Woes can provide customer context and support evidence that make incident triage faster.

## Key design questions

- Which Woes events are safe to page on?
- How should service and escalation policy selection work?
- Should Woes create incidents automatically or require operator confirmation?
- How should incident updates sync back into Woes?
- How should duplicate pages be prevented during repeated support messages?

## Possible first version

Start with an automation action that creates a PagerDuty incident for high-priority issues or SLA breaches, with explicit service selection and duplicate protection.
