# Notion Connector

Notion is a planned customer connector for teams that manage product notes, FAQs, implementation guides, and support playbooks in Notion.

## Status

**Planned connector.** This is not currently a live Woes connector. It should remain disabled until page selection, permission handling, source attribution, rescans, and redaction behavior are implemented.

## What it will do

A Notion connector should bring selected workspace pages into the Woes context layer:

- Import pages or databases selected by an admin.
- Use FAQs, troubleshooting guides, and internal runbooks as support context.
- Preserve source links so operators can inspect the original note.
- Rescan pages when support documentation changes.
- Help identify outdated or weak knowledge-base coverage.

## Product fit

Notion often holds the practical explanations that are missing from formal API specs: edge cases, rollout notes, implementation details, and support scripts. Woes can use that material to improve answers while preserving confidence, citations, and handoff rules.

## Implementation requirements

Before Notion becomes active, Woes needs:

- OAuth install and workspace/page picker.
- Page, database, and child-page import rules.
- Permission checks for internal-only material.
- Source attribution and freshness tracking.
- Manual rescan and stale-content behavior.
- Clear rules for what can be cited to customers versus shown only to operators.

## Possible first version

Start with read-only import of selected pages and child pages, plus manual rescan. Keep database sync and write-back for a later version.
