# Route Map

This page is a high-level route map inside the API Reference tab. It is not a complete public API contract. Use the generated endpoint pages for exact `/api/v1` schemas and runnable examples, and use [Widget Routes](/api/widget-routes) for public widget route boundaries.

## Public REST API

These routes use `Authorization: Bearer <woesk_...>` workspace API keys and are documented in the generated API Reference.

| Route family | Purpose |
| --- | --- |
| `/api/v1/conversations` | List and create API-origin support conversations. |
| `/api/v1/conversations/{conversationId}` | Read or update one workspace conversation. |
| `/api/v1/conversations/{conversationId}/messages` | List or append conversation messages. |
| `/api/v1/issues` | List and create support issues. |
| `/api/v1/issues/{issueId}` | Read or update one support issue. |
| `/api/v1/clients` | List client/account aggregates or create manual client/account seeds. |
| `/api/v1/clients/{clientId}` | Read one client/account aggregate. |
| `/api/v1/agents` | List public agent metadata. |
| `/api/v1/agents/{agentId}` | Read one public agent metadata object. |
| `/api/v1/sources` | List or create API context sources. |
| `/api/v1/sources/{sourceId}` | Read, update, or delete one API context source. |
| `/api/v1/sources/{sourceId}/rescan` | Rescan one URL-backed API context source. |
| `/api/v1/tags` | List managed tag definitions. |
| `/api/v1/statuses` | List issue status definitions. |
| `/api/v1/priorities` | List issue priority definitions. |
| `/api/v1/custom-fields` | List customer custom field definitions. |
| `/api/v1/usage` | Read current-month workspace usage meters. |
| `/api/v1/usage/api-keys` | Read current-month public API calls by key prefix. |
| `/api/v1/analytics` | Read sanitized analytics aggregates. |

## Widget And Customer Routes

These routes are customer-facing, SDK-facing, or webhook-facing. They do not accept `woesk_` workspace API keys.

| Route family | Purpose |
| --- | --- |
| `/api/widget/config` | Public widget presentation config for a workspace or agent key. |
| `/api/widget/identity` | HMAC/JWT identity proof verification for a widget key. |
| `/api/widget/messages` | Customer widget conversation creation, message send, and history. |
| `/api/widget/events` | Widget impression, open, close, and engagement tracking. |
| `/api/widget/presence` | Customer presence heartbeat for an existing widget conversation. |
| `/api/widget/feedback` | Message feedback state for a verified widget conversation. |
| `/api/widget/csat` | Widget CSAT or NPS survey responses. |
| `/api/widget/updates` | Published widget updates and optional verified-reader read state. |
| `/api/widget/updates/read` | Mark widget updates read for a verified reader. |
| `/api/client-portal/auth/request-code` | Request a customer portal magic code. |
| `/api/client-portal/auth/verify-code` | Verify a customer portal magic code and start a portal session. |
| `/api/client-portal/auth/logout` | End a customer portal session. |
| `/api/client-portal/issues` | Customer-safe portal issue list and creation for valid portal sessions. |
| `/api/client-portal/issues/{issueId}` | Customer-safe portal issue detail for valid portal sessions. |
| `/api/client-portal/issues/{issueId}/messages` | Customer portal issue message creation. |
| `/api/client-portal/members` | Customer portal member list and invite flows. |
| `/api/surveys/email-response` | Signed email survey response capture. |
| `/api/email/unsubscribe` | Signed customer email unsubscribe flow. |

## Team Chat Routes

Team Chat routes are authenticated workspace routes for internal teammate messaging. They are listed separately because they are not customer live chat and are not public REST API endpoints.

| Route family | Purpose |
| --- | --- |
| `/api/team-chat/conversations` | List or create DMs and internal conversations. |
| `/api/team-chat/conversations/{conversationId}/messages` | List or send Team Chat messages. |
| `/api/team-chat/conversations/{conversationId}/messages/{messageId}` | Edit or delete a Team Chat message. |
| `/api/team-chat/conversations/{conversationId}/attachments` | List or create Team Chat attachments. |
| `/api/team-chat/conversations/{conversationId}/reactions` | List or add message reactions. |
| `/api/team-chat/conversations/{conversationId}/read` | Mark a Team Chat conversation read. |
| `/api/team-chat/channels` | List joined/discoverable channels or create a channel. |
| `/api/team-chat/channels/{conversationId}` | Update channel settings. |
| `/api/team-chat/channels/{conversationId}/members` | List or add channel members. |
| `/api/team-chat/channels/{conversationId}/members/{userId}` | Remove a channel member. |
| `/api/team-chat/channels/{conversationId}/join` | Join a discoverable channel. |
| `/api/team-chat/channels/{conversationId}/leave` | Leave a channel. |
| `/api/team-chat/channels/{conversationId}/archive` | Archive a channel. |
| `/api/team-chat/channels/{conversationId}/unarchive` | Restore an archived channel. |

See [Team Chat](/platform/team-chat) for product behavior, permissions, and boundaries.

## Webhook Routes

Webhook routes are server-to-server integration endpoints. They do not accept workspace API keys.

| Route family | Purpose |
| --- | --- |
| `/api/stripe/webhook` | Receives Stripe subscription events and syncs billing entitlements. |
| `/api/email/inbound/resend` | Receives inbound email events from Resend. |
| `/api/discord/events` | Receives Discord interaction and message events. |

## Admin Routes

Admin routes under `/api/admin/*` are platform-admin only. They are intentionally not customer API surfaces.

<Note>
  Routes listed here may require authentication, workspace permissions, signed tokens, provider webhooks, public widget keys, conversation secrets, or platform-admin access depending on the route family.
</Note>
