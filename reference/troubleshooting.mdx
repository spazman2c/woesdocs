# Troubleshooting

Use this page when a workspace setup or support workflow does not behave as expected.

## Widget

| Symptom | Check |
| --- | --- |
| Widget does not load | Confirm script URL, public key, CSP, and page installation. |
| Wrong agent answers | Use the correct agent widget key and verify the agent has attached sources. |
| Customer appears anonymous | Add identify calls or configure verified widget identity. |
| History does not load | Confirm stable identity and conversation secret handling. |

## Context

| Symptom | Check |
| --- | --- |
| Source fails to scan | Confirm the URL is reachable and safe to fetch. |
| No endpoints found | Add OpenAPI/Postman/schema material or endpoint reference pages. |
| Agent asks too many clarifying questions | Inspect source coverage and add missing endpoint/auth/error details. |
| Test request fails | Confirm source auth settings and safe endpoint URL. |

## Agents

| Symptom | Check |
| --- | --- |
| Agent skips answers | Review confidence, retrieved context, and source coverage. |
| Agent gives generic answers | Attach more specific sources and run context learning. |
| Agent labels missing in analytics | Confirm agents are configured and runs include agent identity. |

## Surveys

| Symptom | Check |
| --- | --- |
| Widget survey does not appear | Confirm survey channel, trigger timing, and closed/solved conversation state. |
| Email survey does not send | Confirm Resend configuration, verified domain, customer email, and enabled email survey. |
| NPS score is rejected | Confirm the survey type is NPS and score is 0 to 10. |

## Analytics

| Symptom | Check |
| --- | --- |
| A filter is missing | Filters are dashboard-specific; switch dashboards to see relevant controls. |
| Live chat views are low | Some reports infer views from widget opens when launcher views are unavailable. |
| No survey responses | Enable surveys and confirm customers have submitted feedback. |

## Billing

| Symptom | Check |
| --- | --- |
| Manage subscription fails | Confirm Stripe is configured and the workspace has permission to manage billing. |
| Subscription status is missing | Confirm Checkout or Stripe Portal has completed and webhook sync has run. |
| Plan downgrade is blocked | Current features or usage exceed the target plan's limits. |
| Add-on capacity is not reflected | Confirm the Stripe subscription item quantity and wait for the subscription update webhook. |
| Public API usage looks high | Review the Public API by key section in Billing and rotate unused API keys. |

## Bring Your Own AI

| Symptom | Check |
| --- | --- |
| BYO AI settings are hidden | The workspace must be on the Bring Your Own AI plan. |
| Provider key does not verify | Confirm the provider key, account permissions, and selected supported model. |
| Agent still uses platform AI | Activate a verified provider in Settings → Bring Your Own AI. |
| Model changed unexpectedly | Unsupported or stale model ids are coerced to the provider default model. |

## Assignment

| Symptom | Check |
| --- | --- |
| Issue does not route | Confirm assignment rules are enabled and the target team has active members. |
| Nobody is eligible | Check capacity, away/inactive status, and required skills. |
| Skill routing blocks work | Broaden the skill dimension or disable skill-block queueing for that team. |
| Automation overwrote an assignee | Use the automation option to assign only when the issue is unassigned. |

## Notifications

| Symptom | Check |
| --- | --- |
| No browser notification | Confirm browser permission, channel master switch, and event matrix setting. |
| Push is disabled | Confirm the browser supports push and server push configuration is enabled. |
| Sounds do not play | Confirm sound channel is enabled and quiet hours are not suppressing it. |

## Client Portal

| Symptom | Check |
| --- | --- |
| Customer cannot sign in | Confirm portal is enabled, email delivery is configured, and the contact has access. |
| Customer sees no issues | Check access level: Members see their own email's issues; Admins see account-level issues. |
| Custom domain is pending | Verify the displayed DNS records before sharing the domain publicly. |

<Tip>
  If an issue looks like missing data, check the relevant source route or setup page before assuming analytics is broken.
</Tip>
